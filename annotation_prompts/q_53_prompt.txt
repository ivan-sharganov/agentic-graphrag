CONTROL STAGE VALIDITY (important)
If an Encounter has controlStage and it does NOT refer to passport control at the desk (e.g., AtGate / AtCheckin and similar), then do NOT include DocumentCheck from that Encounter.
If controlStage is absent, you may assume it is passport control, but only if the Cypher explicitly has DocumentCheck for that Encounter.

PROCEDURE FOR EACH cypher_i.txt
Step 1. Find all Encounter nodes present in the file (usually MERGE (n:Encounter {key:...})).
Step 2. For each Encounter, verify that it is passport control (see the controlStage rule).
Step 3. Keep only Encounter nodes that can be linked to Italy via atCountry or via atCity/atAirport + locatedInCountry; for each such Encounter keep the connected City (or Airport→City).
Step 4. For valid Encounter nodes, find all DocumentCheck linked to it (Encounter -> hasDocumentCheck -> DocumentCheck).
Step 5. For each DocumentCheck, find its document type (DocumentCheck -> requestedDocument -> DocumentInstance and DocumentInstance.documentType).
Step 6. Build triples minimally:
  A) evidence: DocumentCheck -> requestedDocument -> DocumentInstance
  B) evidence: DocumentInstance -> documentType -> "..."
  C) connection: Encounter -> hasDocumentCheck -> DocumentCheck
  D) scope: Encounter -> atCity / atCountry / atAirport and related locatedInCity / locatedInCountry, if present.

Semantic matching is allowed (e.g., airport ↔ city from the list), even if there is no explicit locatedInCity/locatedInCountry link.
