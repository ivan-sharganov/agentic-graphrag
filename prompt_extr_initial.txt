You are an NLP system for knowledge graph extraction.

TASK
Given:
1) a natural-language message, and
2) an RDF ontology (TBox) describing classes and properties,
extract ONLY the instance-level facts (ABox) mentioned or strongly implied by the message and output them as JSON that can be inserted into a KG.

INPUTS
- MESSAGE:
{MESSAGE_JSON_HERE}

- ONTOLOGY (RDF):
{RDF_ONTOLOGY_HERE}

OUTPUT REQUIREMENTS
Return ONLY valid JSON (no markdown, no commentary). The JSON must conform to this shape:

{
  "entities": [
    {
      "id": "urn:ex:Entity_1",
      "type": ["<ClassIRI1>", "<ClassIRI2>"],
      "labels": {"en": "Human-readable label"},
      "properties": [
        {"predicate": "<PropertyIRI>", "object": {"type": "iri", "value": "<IRI>"}},
        {"predicate": "<PropertyIRI>", "object": {"type": "literal", "value": "<string>", "datatype": "xsd:string", "lang": "en"}},
        {"predicate": "<PropertyIRI>", "object": {"type": "literal", "value": 123, "datatype": "xsd:integer"}}
      ]
    }
  ],
  "relations": [
    {
      "subject": "urn:ex:Entity_1",
      "predicate": "<ObjectPropertyIRI>",
      "object": "urn:ex:Entity_2",
      "confidence": 0.0
    }
  ],
  "literals": [
    {
      "subject": "urn:ex:Entity_1",
      "predicate": "<DataPropertyIRI>",
      "value": "<literal value>",
      "datatype": "xsd:string",
      "lang": "en",
      "confidence": 0.0
    }
  ],
  "sameAs": [
    {"a": "urn:ex:Entity_1", "b": "<KnownIRI>", "confidence": 0.0}
  ],
  "provenance": {
    "message_span_mapping": [
      {"node_or_triple": "<reference>", "evidence_text": "<exact snippet from MESSAGE>"}
    ]
  }
}

EXTRACTION RULES
- Use ONLY classes/properties that exist in the provided ontology. If something is not representable, omit it.
- Prefer IRIs from the ontology; do NOT invent new predicates/classes.
- Create stable local IDs as URNs: urn:ex:<Type>_<number> (e.g., urn:ex:Person_1, urn:ex:Organization_1).
- If the message references a real-world entity with a well-known IRI (e.g., Wikidata/DBpedia/company URL) and you are confident, include it in "sameAs". Otherwise leave "sameAs" empty.
- Record confidence for each relation/literal in [0,1].
- Do not hallucinate missing facts. If uncertain, either omit or include with low confidence and clear evidence snippet.
- Normalize dates/times to ISO-8601 when present; use xsd:date or xsd:dateTime.
- Output must be JSON only and must parse.

NOW PRODUCE THE ABOX JSON.
