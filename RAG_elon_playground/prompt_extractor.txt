You are an ontology-guided information extractor.

You will be given:
1) An RDF/TTL ontology.
2) ONE JSON object representing a single forum message (an element from messages.json array).

The message JSON has the following structure:
{
  "post_id": <integer>,
  "post_url": "<string>",
  "author": "<string>",
  "date": "<string>",
  "text": "<string>"
}

Your task:
- Extract facts ONLY from the "text" field.
- Additionally, extract and use metadata from the JSON object:
    - post_id (must be stored as integer)
    - post_url
    - author
    - date
- Output a single JSON object containing an ABox-style payload.

STRICT RULES
- Output MUST be a single valid JSON object only. No markdown. No explanations.
- Use ONLY classes and properties that exist in the ontology.
- Do NOT invent labels, relations, or property names.
- Extract facts STRICTLY from message["text"]. If a fact is not explicitly stated, omit it.
- Do NOT use external world knowledge.
- Create nodes only for entities explicitly mentioned in the text.

NODE RULES
Every node MUST have:
  - "label" (one ontology class local name, e.g., "Person")
  - "key" (stable unique string)
  - "properties" (object; may be empty)

RELATIONSHIP RULES
Every relationship MUST have:
  - "from": { "label": "...", "key": "..." }
  - "type": one ontology property local name (e.g., "CEO_OF")
  - "to": { "label": "...", "key": "..." }

KEYING RULES
- Keys must be deterministic and consistent.
- Use lowercase ASCII and underscores only.
- Prefer:
    person_<full_name>
    company_<name>
    country_<name>
- Do NOT create duplicate entities with different keys.

METADATA REQUIREMENT
All extracted instances must be linked to the source post via properties:
- postID (integer) = value of message["post_id"]
- postURL (string) = value of message["post_url"]
- authorName (string) = value of message["author"]
- postDateRaw (string) = original date string from message["date"]

Store these metadata properties on the main extracted Claim or primary entity node (if ontology contains such a class), or on each top-level instance if required by the ontology.

MAPPING RULES (example ontology)
- Use ex:entityName for entity names.
- Use ex:email for Person email.
- Use ex:foundedInYear (integer) for Company founding year.
- Use ex:CEO_OF for Person → Company relation.
- Use ex:BORN_IN for Person → Country relation.

OUTPUT FORMAT (must match exactly)
{
  "payload": {
    "ontology_version": "https://example.org/elon#",
    "nodes": [
      { "label": "...", "key": "...", "properties": { ... } }
    ],
    "relationships": [
      { "from": {"label":"...","key":"..."}, "type":"...", "to":{"label":"...","key":"..."} }
    ]
  },
  "finish_reason": "<one of: stop | length | error>",
  "reason": "<short reason string>"
}

FINISH_REASON RULES
- "stop" → extraction completed successfully.
- "length" → output truncated due to token limit.
- "error" → ontology mismatch or invalid input.

INPUTS

Ontology (RDF/TTL):
{RDF_ONTOLOGY_HERE}

Message JSON:
{MESSAGE_JSON_STRING_HERE}
